## Overview

This project implements a Discrete Event Simulation (DES) of an web server
environment. The simulation
models a server with closed-loop users, thread-per-task scheduling,
round-robin core assignment, context-switch overhead, and request timeouts.

It produces all performance metrics (response time with 95% CI, goodput, badput,
throughput, drop rate, core utilization) and compares simulation results against
both the MVA analytic model and real measured data from Assignment 1.

## How to Build and Run

### Step 1 Compile the simulation

```bash
g++ web_des_sim.cpp -o web_sim
```

### Step 2 Run the simulation

```bash
./web_sim
```

This takes approximately **4–6 minutes** on a modern machine. It runs:
- 120 simulation runs (12 user counts × 10 replications) for the main experiment
- 30 runs for the core-count experiment
- 30 runs for the service-distribution experiment
- 1 run for the Welch warmup trace

When complete it writes four CSV files:
`results.csv`, `results_comparison.csv`, `results_cores.csv`, `welch_trace.csv`


### Step 3 Generate all graphs

```bash
python3 plot_graphs.py
```

Reads the four CSV files and writes 16 PNG files to `./graphs/`.
Prints a confirmation line for each graph as it saves.

## System Configuration (Calibrated from Assignment 1)

| Parameter | Value | Source |
|---|---|---|
| Server cores | 1 (Apache pinned to Core 0)
| Service time | Exponential, mean = 31 ms | Utilisation Law: S = 1/X_sat = 1/32 |
| Think time |  mean = 4 s | Little's Law: Z = N/X − R |
| Timeout | 5 + Exp(5) s, mean = 10 s | Chosen to reproduce ~0 drops |
| Context-switch | Exp(mean = 2 ms) | Standard OS overhead |
| Scheduling | Round-robin across cores | Assignment requirement |
| User counts | {20,30,40,50,60,70,80,100,120,140,150,160} | Matches Assignment 1 tsung points |
| Replications | 10 per user count | For 95% CI computation |
| Simulation duration | 10,000 s per replica | Statistical sufficiency |
| Warmup discarded | First 1,000 s (Welch's method) | Transient removal |
| Measurement window | 9,000 s per replica | Actual stats collection window |

## Output Files

### CSV files (produced by web_des_sim)

**results.csv**
Main experiment results. One row per user count.
Columns: `users, avg_rt, rt_lo, rt_hi, goodput, badput, throughput, drop_rate_pct, core_util_pct`

**results_comparison.csv**
Three-way comparison for Measured vs MVA vs Simulation.
Columns: `users, measured_rt, mva_rt, sim_rt, measured_x, mva_x, sim_x, mva_util_pct`

**results_cores.csv**
experiment: effect of core count at N=100.
Columns: `cores, avg_rt, rt_lo, rt_hi, goodput, drop_rate_pct, core_util_pct`

**welch_trace.csv**
Columns: `time, running_avg_rt_ms`

### Input file

**measured_data.csv**
Real data from Assignment 1 tsung closed-loop experiment.
Columns: `users, avg_rt, throughput`
